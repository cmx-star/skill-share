# 自然语言检查清单

## 常见问题

- **空话开场或收尾**：删掉后不影响信息，就直接删，不要换成另一句空话。
- **抽象名词堆叠**：把“提升协同效率”改成谁要做什么、结果是什么；原文没有事实时，不要编。
- **句子绕路**：删掉“需要注意的是”“这意味着”“从而实现”等提示层，直接说结论或事实。
- **结构过于工整**：不要为了“首先、其次、最后”硬凑三点；结构应服务逻辑。
- **语域混杂**：技术规则保持准确，营销语保持克制，别在同一段里来回切换。
- **把读者当外行**：不要重复解释显而易见的事，也不要用空泛鼓励代替答案。

## 改写方法

优先使用明确的主语和动词。下面是方向，不是机械替换表：

| 生硬表达 | 更自然的写法 |
| --- | --- |
| 对当前任务进行上下文恢复 | 恢复工作时，先确认当前任务的已知信息 |
| 将结果纳入当前工作 | 把返回结果处理完，再继续下一步 |
| 仅在满足条件时执行 | 只有满足这个条件时才执行 |

## 交稿前回读

1. 每个事实、数字、术语和限制条件是否还在？
2. 读者能否一遍看懂该做什么？
3. 是否删掉了空话，却没有补进新的空话？
4. 正式内容是否仍然正式，技术内容是否仍然准确？
5. 有没有为了显得自然而加入原文没有的判断、承诺或细节？
